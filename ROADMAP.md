# Roadmap

Vouch is being built incrementally, in public, with a focus on usefulness at every stage. This document describes where we are and where we're headed.

---

## Vision

A world where any business — regardless of size or technical resources — can understand and act on the data they already have. Vouch aims to be the simplest possible path from raw business data to clear, actionable revenue insights.

---

## Guiding Principles

- **Ship early, ship often** — each milestone should be genuinely useful on its own
- **Don't require infrastructure** — if it needs a data engineer to set up, it's too complex
- **Trust through transparency** — users should be able to understand how every insight was reached. In practice, this means insights today are generated by deterministic, readable functions in `lib/insights/` — not by an AI model — so every number and recommendation traces back to code anyone can read
- **Community-driven** — features are prioritized by real-world need, not roadmap theater

---

## Milestones

### v0.1 — Foundation (Community Preview)

The first usable version. Focused entirely on making the core workflow work reliably.

- [x] Landing page
- [x] Upload a CSV file (drag-and-drop or browse, entirely client-side)
- [x] Auto-detect column names, with an editable mapping confirmation step
- [x] Summary dashboard with key metrics, charts, and tables
- [x] Business Health strip and a plain-English Executive Summary
- [x] Bundled sample dataset and a "Try with sample data" demo mode
- [x] Export the summary as a `.txt` file, copy it, or print it
- [ ] Deeper revenue-opportunity insights (lapsed customers, pricing gaps, anomalies) beyond the current Business Health / Executive Summary observations
- [ ] Export the full underlying dataset as a summary `.csv` (today's export is a text summary, not a CSV)

**Goal:** Someone should be able to upload a sales export and get a useful insight in under five minutes. Largely achieved for the "upload → insight" path; the deeper insight types above are still open.

---

### v0.2 — Intelligence (Planned)

Making the results smarter and more actionable, while staying deterministic and transparent.

- [ ] Additional insight types: leads needing follow-up, revenue at risk, conversion bottlenecks, high-value inactive customers, slow-moving deals
- [ ] Smarter, more configurable field mapping (custom synonyms, saved mappings)
- [ ] Saved sessions (revisit a previous analysis without re-uploading)
- [ ] Richer charts and visualizations

**Goal:** Results should feel tailored to your data, not generic. (An earlier draft of this roadmap mentioned AI-generated recommendations; the project has since deliberately committed to deterministic, rule-based logic instead, to keep every insight explainable. That direction is now a guiding principle, not just a v0.2 detail.)

---

### v1.0 — Community Edition Vision

A stable, documented release that teams and consultants can rely on. Nothing below has started yet — this section describes intent, not commitments.

- [ ] Stable, versioned public API
- [ ] Self-hosted deployment guide (Docker, cloud options)
- [ ] Multi-file and multi-source support
- [ ] Community plugin system for custom insight types
- [ ] Full documentation site

**Goal:** Production-grade reliability that organizations can build on.

---

## Beyond v1.0

Ideas being considered for future releases — not committed, subject to community input:

- Native integrations (Shopify, Stripe, QuickBooks exports)
- Scheduled analysis runs
- Team collaboration features
- Role-based access for self-hosted deployments

---

## How to Influence the Roadmap

This roadmap is a living document. If you have a use case that isn't represented here, open a [GitHub Issue](../../issues) and describe it. The best way to move something up the list is to show that real people need it.
