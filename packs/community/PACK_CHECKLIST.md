# Pack contribution checklist

- [ ] Lowercase stable id, version, category and maintainer
- [ ] Clear field labels, importance and non-ambiguous aliases
- [ ] Ordered stages with explicit lifecycle semantics
- [ ] Stable versioned rule ids
- [ ] Evidence requirements and missing-evidence copy
- [ ] Synthetic sample data with no customer information
- [ ] Pack validation, mapping, rule and comparison tests
- [ ] Stable example route or documented preview
- [ ] Privacy and exported-field review
- [ ] Maintainer/contributor acknowledgement
